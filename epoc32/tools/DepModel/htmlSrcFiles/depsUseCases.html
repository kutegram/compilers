Static Dependency Use Cases

Here you can find information on some typical use cases of the DepModel static dependency tool. DepModel home page is here.

Use case index

The use cases for Executables, Components and Subsystems are very similar. For convenience they are listed separately here.

Uses cases for executables:

  • UC.Exec.1: Find textual information on an arbitrary executable
  • UC.Exec.2: Find the dependency diagram for an arbitrary executable
  • UC.Exec.3: Find out why an executable depends on one or more others (using imports)
  • UC.Exec.4: Find out why several executables depend on another executable (using exports)

Use cases for components:

  • UC.Comp.1: Find textual information on an arbitrary component
  • UC.Comp.2: Find the dependency diagram for an arbitrary component
  • UC.Comp.3: Find out why a component depends on one or more others (using imports)
  • UC.Comp.4: Find out why several components depend on another component (using exports)

Use cases for subsystems:

  • UC.Subsys.1: Find textual information on an arbitrary subsystem
  • UC.Subsys.2: Find the dependency diagram for an arbitrary subsystem
  • UC.Subsys.3: Find out why a subsystem depends on one or more others (using imports)
  • UC.Subsys.4: Find out why several subsystems depend on another subsystem (using exports)

Other use cases:

  • UC.General.1: Find a list of all executables sorted by component and subsystem

UC.Exec.1: Find textual information on an arbitrary executable

Concrete case: Find information on cntmodel.dll [shortcut: deps/Executables/cntmodel.dll.html]

  1. navigate to the index of all executables, using the hyperlink EXECUTABLES INDEX at top of each page (including this page)
  2. use the alphabetic links at the top of the EXECUTABLES INDEX page to find cntmodel.dll on the index page
  3. follow the link to cntmodel.dll

The table at the top of the page lists:

  • Purpose of the executable (if documented)
  • Subsystem/Component of the executable
  • Target path
  • Sizes
  • Build files (for finding headers)
  • Make file (for finding source code)

The rest of the page provides dependency information:

  • Exports: the list of executables cntmodel.dll exports to
  • Imports: the list of executables cntmodel.dll imports from
  • Dependency tree: the tree.

UC.Exec.2: Find the dependency diagram for an arbitrary executable

Concrete case: Find the diagram for cntmodel.dll

  1. navigate to cntmodel.dll's textual dependency page using use case UC.Exec.1
  2. click on the 'Graphical representation' hyperlink near the top of the text page.

UC.Exec.3: Find out why an executable depends on one or more others (using imports)

Concrete case: Find out why cntmodel.dll depends on bafl.dll and on versit.dll

  1. navigate to cntmodel.dll's textual dependency page using use case UC.Exec.1
  2. scroll down to cntmodel.dll's imports
  3. click on the magnifying glass icon next to the link to bafl.dll's text page to drill down to the list of functions that cntmodel.dll imports from bafl.dll
  4. go back to the previous page
  5. click on the magnifying glass icon next to the link to versit.dll's text page to drill down to the list of functions that cntmodel.dll imports from versit.dll.

UC.Exec.4: Find out why several executables depend on another executable (using exports)

Concrete case: Find out why cntvcard.dll and contacui.dll depend on cntmodel.dll

  1. navigate to cntmodel.dll's textual dependency page using use case UC.Exec.1
  2. scroll down to cntmodel.dll's exports
  3. click on the magnifying glass icon next to the link to cntvcard.dll's text page to drill down to the list of functions that cntvcard.dll imports from cntmodel.dll
  4. go back to the previous page
  5. click on the magnifying glass icon next to the link to contacui.dll's text page to drill down to the list of functions that contacui.dll imports from cntmodel.dll.

UC.Comp.1: Find textual information on an arbitrary component

Concrete case: Find information on component DBMS [shortcut: deps/Components/dbms.html]

  1. navigate to the index of all components, using the hyperlink COMPONENTS INDEX at top of each page (including this page)
  2. use the alphabetic links at the top of the COMPONENTS INDEX page to find DBMS on the index page
  3. follow the link to DBMS

The table at the top of the page lists:

  • Description of the component (if documented)
  • Subsystem of the component
  • Build files (for finding headers)
  • A list of the executables contained by the component.

The rest of the page provides dependency information:

  • Exports: the list of components DBMS exports to
  • Imports: the list of components DBMS imports from
  • Dependency tree: the tree.

UC.Comp.2: Find the dependency diagram for an arbitrary component

Concrete case: Find the diagram for component DBMS

  1. navigate to DBMS component's textual dependency page using use case UC.Comp.1
  2. click on the 'Graphical representation' hyperlink near the top of the text page.

UC.Comp.3: Find out why a component depends on one or more others (using imports)

Concrete case: Find out why DBMS component depends on STORE (or F32 or E32).

  1. navigate to DBMS component's textual dependency page using use case UC.Comp.1
  2. scroll down to DBMS's imports
  3. click on the magnifying glass icon to drill down to the list of executables that DBMS's executables import from
  4. find the entries for STORE (or F32 or E32) using your browser's 'Find on page' option
  5. click on the magnifying glass icon to drill down to the list of functions that each executable in DBMS imports from executables in STORE (or F32 or E32).

UC.Comp.4: Find out why several components depend on another component (using exports)

Concrete case: Find out why CNTMODEL and COMMDB depend on DBMS

  1. navigate to DBMS component's textual dependency page using use case UC.Comp.1
  2. scroll down to DBMS's exports
  3. click on the magnifying glass icon next to the link to CNTMODEL's text page to drill down to the list of DBMS's executables that CNTMODEL imports from
  4. find the entries for DBMS using your browser's 'Find on page' option
  5. click on the magnifying glass icon to drill down to the list of functions that each executable in CNTMODEL imports from DBMS
  6. go back to the DBMS export list to repeat the same steps for COMMDB.

  3. click on the magnifying glass icon to drill down to the list of executables that COMMS-INFRAS's executables import from
